介绍与个人感想

这门课是 UC Berkeley EECS 的编程入门课，主要使用 Python 语言，也介绍了 Scheme 与 SQL，我想意在将 Python 与函数式与描述式的语言作比较，同时也以此介绍了编程范式与数据库。

这门课由于改编于 MIT 基于 Scheme 的 SICP，其中 Python 语言的思维也十分函数式，绝大多数讲义与作业都是在写函数，并用交互式环境进行互动。且正如标题所言，这门课对 Python 语言的刨析很深，讲了很多所谓的 Evaluation Rule 与其他很多自己瞎学中较少关注到的信息（学了这门课之后我总算明白为什么教人编程时别人都会犯那么多奇奇怪怪的错，其实就是这些可能犯的错被我们编程的 “经验主义” 所解决了），出于反对经验主义的目的，这门课也是非常值得学习的。

作为一个懒狗与暂时的速通需求，我并没有做 Discussions 和 Exam （要画太多 Environment Diagram 了），除了 Scheme Interpreter 外也暂时并没有做 Projects （不急）

Scheme Interpreter 这个 Project 我认为是整门课最精华的 Project，因为这是既从上而下又从下而上地解释这门课的标题”Structure and Interpretation”，同时也引入了一些编译原理的概念，我认为就算是对于速通来说也是必做的。

Doctest

  • 嵌入到源码中 / 独立文件
    • 源码中可存放的位置：模块最开头 / 函数声明语句的下一行
  • 执行：python3 -m doctest (-v) filename.py
    • 其中可选的 -v 为 verbose=True，即显示详细 doctest 过程

Referential Transparency:

引用透明性：一个表达式在给定程序中是引用透明的，如果对于任何表达式，所有的实例都可以被其计算结果所替代，而不改变程序的行为。这意味着，如果一个函数对于相同的输入总是产生相同的结果，并且在执行过程中不产生任何可观察的副作用（如修改全局变量、进行 I/O 操作等），那么这个函数就是引用透明的。
