Finite State Machines

Very Simple Computers

  • Computation as machinery: Finite Automata (abbr. FA 有限自动机) or Finite State Machines
    • The machine will just move from one state to another
    • There are finitely many states
    • Which state we move to depend on input: a symbol drawn from a finite alphabet
    • Some states are accepting
    • There’s an identified start state.
    • 自动机是一个对信号序列进行判定的数学模型

FAs with accepting states

  • The double circle is called accepting state or final state. The set of accepting states is
  • If the automation ends in a accepting state, we say the automata has accepted the string. (babaaba)
    • If the automation ends in a non-accepting state, we say the automata has rejected the string. (babbaa)

Space saving conventions

  • The bottom state is called black hole state: once there, the machine never leaves.
    • The black hole convention says that if you don’t write a transition for letter a from state q, there is an implicit a-transition from q to a non-accepting black hole state.

在自动机理论中，黑洞约定可以被视为一种节省空间的技巧，通过定义一个黑洞来减少表示各个状态 /state 的输入。

Non-determinism and Regular Expressions

Non-detertminism

  • A NFA may have:
    • any number of start states
    • any number of transitions for each letter from each state
  • Formally, An NFA comprises:
    • A finite set Q of states
    • A finite alphabet Σ of input letters
    • A transition relation
    • A set of starting states
    • A subset of accepting states

NFA behaviour

Parallel understanding

  • An NFA may have many active states
    • All the start states are initially active
    • whenever q is active, if input a occurs, then all become active instead of q
    • at end of input, the NFA accepts if any active state is accepting.
  • Since one bit for a state, running an NFA requires exponentially more memory than a DFA

Non-detertminism understanding

  • At the beginning, the machine chooses a start state out of S
